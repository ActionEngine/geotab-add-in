**Introduction**

Supported Driver Change Types.

**Properties**

## Driver

User-based Driver. Makes a[Device](../Device/index.md)continuously associated with particular[Driver](../Driver/index.md)until a different driver id is assigned to the same device. First record in a trip overrides any other record encountered during the trip it was added to, except "TripDriver".

## DriverKey

Driver Key - based Driver. Drivers use this key type to identify themselves. This will typically be used when the driver ID relay is enabled. Makes a[Device](../Device/index.md)continuously associated with a particular[Driver](../Driver/index.md)until a different driver ID is assigned to the same device.

## DriverVehicleChange

DriverVehicleChange log - based Driver. Auto-generated by the system to support the driver ID relay feature.

## None

Unknown key type.

## ResetDriver

System-detected Driver reset (to UnknownDriver). Makes a[Device](../Device/index.md)permanently disassociate from the previous[Driver](../Driver/index.md)due to concurrent permanent registration of the same Driver for a different device.

## TripDriver

User-based Driver that is only active for the duration of a single trip.
